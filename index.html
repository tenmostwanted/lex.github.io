阿里云体验

阿里云OSS（Object Storage Service）是阿里云提供的一种基于互联网的数据存储服务。它允许用户通过简单的接口存储和管理任意类型的数据。OSS 支持海量数据的存储和高并发访问，提供了高可用性和安全性，适合存储大规模的非结构化数据，如图片、音频、视频、备份文件等。

（一）了解阿里云OSS服务

阿里云OSS是一种对象存储服务，具有以下特点：

  • 对象存储：支持海量数据存储，数据以对象形式存储。
  • 高可用性与持久性：提供99.9999999999%的数据持久性，适合长期存储和备份。
  • 数据分级存储：根据数据访问频率可选择标准存储、低频存储和归档存储，优化成本。
  • 全网加速访问：通过CDN与全球加速点，用户可以实现全球范围内的低延迟访问。

（二）对比ECS本地存储、OSS、NAS存储服务的特点

存储类型 ECS本地存储 OSS（对象存储服务） NAS（网络附加存储）
存储类型 块存储 对象存储 文件存储
数据结构 适用于操作系统和应用的块级数据访问，类似于本地硬盘 适合存储非结构化数据（图片、音频、视频等） 共享文件系统，适合需要多个ECS实例共享文件
典型使用场景 操作系统、数据库等需要高I/O性能的场景 静态资源（图片、视频、备份文件）存储、备份和归档 大型网站、分布式应用，共享文件存储，如日志管理、文件共享
数据访问方式 直接挂载为虚拟磁盘，块设备访问 RESTful API访问，基于HTTP/HTTPS 通过标准文件协议（NFS/SMB）挂载，文件系统访问
性能 高I/O性能，适合低延迟和高读写场景 高吞吐量，适合海量数据的高并发访问 文件共享，适合中等I/O需求和多实例共享访问
持久性 与ECS实例生命周期相关 高持久性，数据冗余存储 高持久性，冗余备份
费用 随ECS实例按量付费 根据存储容量和流量计费，价格灵活 按实际使用的存储空间和流量计费
扩展性 受限于实例配置 高扩展性，支持海量存储 支持自动扩展

（三）三种存储服务的适用应用场景

  1. ECS本地存储：

    • 适用于对高性能和低延迟有要求的场景，例如数据库、操作系统及应用的主存储。
    • 常用于高I/O密集型工作负载，如实时分析、事务处理、数据库应用等。

  2. OSS（对象存储服务）：

    • 适合静态数据存储，如图片、视频、音频、日志文件、备份数据等。
    • 常用于内容分发、静态网站托管、大数据存储与处理、云上备份和归档。

  3. NAS（网络附加存储）：

    • 适合需要多台ECS实例共享同一文件系统的场景，如日志存储、共享文件系统、开发环境或分布式应用。
    • 常用于企业应用共享、数据分析任务和容器化应用存储等。

（四）实验结论或体会

  1. 存储服务的选择应基于应用的特点和需求：

    • 对于需要高性能和低延迟的应用，如数据库和操作系统，ECS本地存储是首选。
    • 如果应用需要海量数据存储和高并发访问，如备份文件和媒体资源分发，OSS 对象存储更具优势。
    • 在需要共享文件系统的场景，如多个应用实例需要访问相同的数据，NAS 是一个合适的选择。

  2. 成本与性能的权衡：

    • 不同的存储服务在成本上有显著差异。OSS提供多种存储类型（如低频存储、归档存储），可按需选择，降低成本。而ECS本地存储和NAS则更适合高I/O性能要求的场景，但费用较高。

  3. 云服务的灵活性和扩展性：

    • 阿里云存储服务通过按需扩展、按量计费的模式，提供了很大的灵活性。特别是OSS，支持海量存储的弹性扩展，适用于对数据存储量和访问需求不断增长的业务。

  4. 应用场景驱动存储架构选择：

    • 根据具体应用场景选择适合的存储服务能显著提高效率和降低成本。例如，电商平台可以用OSS存储用户上传的图片和视频，用ECS本地存储运行数据库系统，用NAS实现日志存储和分析。

通过实验，我们可以更深入地理解不同存储服务的特点，从而在实际生产环境中根据需求做出最优选择。

Mysql是怎样存储的

进入Docker Mysql，用命令行查看Mysql的存储目录

发现都是存储在var/lib/mysql这个地方

MySQL 的 NULL 值是怎么存放的？

row的左边（变长字端长度+Null列表）｜ 真实数据

在MySQL表中，NULL 并不占用实际的空间来存放具体的数值，而是通过一个“NULL标志位”来标识字段的值是否为 NULL。这些标志位是和数据一起存储的。

在比较操作中，NULL 被视为一个特殊的值，无法与其他数值进行直接比较（包括 NULL 本身）。使用 IS NULL 或 IS NOT NULL 来判断 NULL。

•	在索引中，如果字段允许为 NULL，那么 NULL 也会被视为索引值的一部分。 在B-Tree索引或Hash索引中，NULL 被处理为一个特殊的值。

varchar（n）中n最大是多少？

最大长度为 65535 字节，这是由于 MySQL 的行最大长度限制。

需要注意的是，n 是字符个数，不是字节数。实际存储的字节数还取决于所使用的字符集。

当然了，还得考虑，变长字段的额外开销，1字节记录Null值，2字节记录变长字段的数量。

如果是UTF-8的，一个字符占3个字节。就是（65535-2-1）/3=21844。

行溢出后，Mysql是如何处理的？

对于InnoDB而言，会放入溢出页。

而不是将其存储在B-Tree叶子节点中。溢出页的结构和B-Tree叶子节点相同，都是由指针数组和溢出数据组成的。

PS. 在真实存放数据的地方会用20个字节的指针指向溢出页，以找到溢出后的数据。
